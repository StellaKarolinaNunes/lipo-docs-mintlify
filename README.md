## Sobre o projeto

A **Documentação LiPo** é uma plataforma de referência dedicada à linguagem **LiPo — Linguagem em Portugol**.

A LiPo foi projetada para apoiar o ensino de lógica de programação por meio de palavras-chave em português, tornando conceitos fundamentais mais acessíveis para estudantes que estão iniciando sua jornada na programação.

Esta documentação centraliza informações sobre sintaxe, variáveis, operadores, estruturas de controle, funções, módulos, exemplos práticos, gramática formal e recursos da linguagem.

O projeto foi construído utilizando **Mintlify**, com conteúdos estruturados em Markdown e MDX para facilitar a leitura, manutenção e expansão da base de conhecimento.

> A Documentação LiPo possui finalidade educacional e busca servir como material de apoio para estudantes, professores e pessoas interessadas em lógica de programação com Portugol.

---

## Objetivo

O objetivo da documentação é oferecer uma fonte organizada, clara e acessível para aprendizagem e consulta da linguagem LiPo.

A plataforma busca reduzir a necessidade de procurar informações em materiais dispersos, reunindo em um único ambiente conteúdos essenciais sobre a linguagem.

Entre os objetivos do projeto estão:

* facilitar o aprendizado de lógica de programação;
* documentar a sintaxe da linguagem LiPo;
* apresentar exemplos práticos de código;
* apoiar estudantes e educadores;
* organizar regras gramaticais e estruturas da linguagem;
* disponibilizar guias de instalação e início rápido;
* documentar o editor online da LiPo;
* incentivar contribuições para evolução da documentação.

## Conteúdos disponíveis

A documentação reúne conteúdos voltados para iniciantes e pessoas que desejam consultar a linguagem LiPo.

* **Início rápido:** primeiros passos para começar a utilizar a linguagem.
* **Instalação:** guia para configurar ferramentas e ambiente necessário.
* **Variáveis e constantes:** declaração, tipos e utilização de dados.
* **Operadores:** operadores aritméticos, relacionais, lógicos e de atribuição.
* **Estruturas de controle:** condicionais, decisões e repetições.
* **Funções:** declaração, parâmetros, retorno e reutilização de código.
* **Sistema de imports:** organização e reutilização de módulos.
* **Comentários e bibliotecas:** recursos complementares da linguagem.
* **Gramática formal:** regras léxicas e sintáticas da LiPo.
* **Exemplos práticos:** códigos para apoiar a aprendizagem.
* **Editor online:** informações sobre o ambiente web da linguagem.
* **Contribuição e licença:** orientações para colaboradores da documentação.

## Como executar localmente

### Pré-requisitos

Antes de iniciar, é necessário ter instalado:

* Node.js em versão LTS;
* npm;
* Git;
* navegador atualizado;
* editor de código opcional, como VS Code;
* conexão com a internet para instalação de pacotes.

### 1. Clone o repositório

```bash
git clone https://github.com/StellaKarolinaNunes/lipo-docs-mintlify.git
```

### 2. Acesse a pasta do projeto

```bash
cd lipo-docs-mintlify
```

### 3. Instale a CLI do Mintlify

```bash
npm install -g mintlify
```

### 4. Execute a documentação localmente

```bash
mintlify dev
```

### 5. Abra no navegador

Após iniciar o servidor, acesse o endereço exibido no terminal.

Normalmente, o Mintlify disponibiliza a documentação em um endereço local semelhante a:

```text
http://localhost:3000
```

---

## Alternativa sem instalação global

Também é possível executar o Mintlify sem instalar a CLI globalmente:

```bash
npx mintlify dev
```

Essa opção é útil para evitar instalações globais no sistema.

## Como adicionar uma nova página

### 1. Crie um arquivo `.mdx`

Exemplo:

```bash
pages/exemplos_praticos/novo_exemplo.mdx
```

### 2. Adicione um título e conteúdo inicial

```mdx
---
title: Novo Exemplo
description: Exemplo prático utilizando a linguagem LiPo.
---

# Novo Exemplo

Aqui será apresentado um novo conteúdo da documentação.
```

### 3. Atualize o arquivo `docs.json`

Inclua a página criada na estrutura de navegação da documentação.

### 4. Execute localmente

```bash
mintlify dev
```

Verifique se a nova página aparece corretamente no menu e se os links internos estão funcionando.

## Contribuição

Contribuições são bem-vindas e ajudam a tornar a documentação mais útil para estudantes, educadores e pessoas que estão aprendendo lógica de programação.

### Processo de contribuição

```bash
# Faça um fork do repositório no GitHub

# Clone o seu fork
git clone https://github.com/SEU-USUARIO/lipo-docs-mintlify.git

# Acesse a pasta do projeto
cd lipo-docs-mintlify

# Crie uma branch para sua alteração
git checkout -b docs/nova-pagina

# Execute a documentação localmente
npx mintlify dev

# Adicione as alterações
git add .

# Crie um commit descritivo
git commit -m "docs: adiciona nova página sobre funções"

# Envie a branch para o GitHub
git push origin docs/nova-pagina
```

Depois, abra um Pull Request explicando:

* qual conteúdo foi adicionado ou atualizado;
* quais páginas foram modificadas;
* se houve alteração no menu de navegação;
* se novas imagens foram incluídas;
* como a documentação foi validada localmente.

### Diretrizes

* Escreva em português claro, objetivo e acessível;
* Utilize títulos e subtítulos bem organizados;
* Prefira exemplos curtos e didáticos;
* Mantenha consistência com os termos utilizados na linguagem LiPo;
* Evite blocos extensos de texto sem divisão;
* Verifique links internos antes de abrir um Pull Request;
* Atualize o `docs.json` ao criar novas páginas;
* Utilize imagens otimizadas e com descrições alternativas;
* Preserve a organização atual de pastas;
* Consulte o arquivo [CONTRIBUTING.md](./CONTRIBUTING.md) para diretrizes detalhadas.
